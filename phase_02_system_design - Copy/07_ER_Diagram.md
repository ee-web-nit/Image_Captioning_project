# Entity Relationship (ER) Diagram

## Project Title

**AI-Powered Image Captioning System Using CNN-Transformer Architecture**

---

# 1. Purpose

The Entity Relationship (ER) Diagram models the logical structure of the database used by the AI-Powered Image Captioning System. It defines the entities, their attributes, and the relationships between them.

Although the core functionality of the system relies on deep learning models, the database plays an important role in storing prediction history, managing trained models, and supporting application analytics.

---

# 2. Objectives

The objectives of the ER Diagram are to:

* Define the database schema.
* Identify entities and their relationships.
* Support efficient data storage and retrieval.
* Provide a blueprint for PostgreSQL database implementation.

---

# 3. Entities

## 3.1 User

Represents users who interact with the application.

### Attributes

* user_id (Primary Key)
* username
* email
* created_at

---

## 3.2 Prediction

Stores every image caption generated by the system.

### Attributes

* prediction_id (Primary Key)
* user_id (Foreign Key)
* model_id (Foreign Key)
* image_name
* generated_caption
* confidence_score
* prediction_time

---

## 3.3 ML Model

Stores information about deployed machine learning models.

### Attributes

* model_id (Primary Key)
* model_name
* model_version
* training_date
* evaluation_score
* deployment_status

---

# 4. Relationships

### User → Prediction

One User can generate many Predictions.

Relationship:

**1 : N**

---

### ML Model → Prediction

One trained model can generate many Predictions.

Relationship:

**1 : N**

---

# 6. Database Design Considerations

The database is normalized to reduce redundancy while maintaining efficient query performance. Foreign key relationships ensure referential integrity between users, predictions, and machine learning models.

Indexes can be created on:

* prediction_time
* user_id
* model_id

to improve search performance and reporting.

---

# 7. Benefits

The ER Diagram provides several advantages:

* Clearly defines database structure.
* Simplifies database implementation.
* Supports scalable prediction storage.
* Enables future extensions such as user authentication, model versioning, and analytics dashboards.
* Improves maintainability of the application.

---

# 8. Conclusion

The Entity Relationship Diagram provides the logical foundation for the PostgreSQL database used by the AI-Powered Image Captioning System. By modeling users, predictions, and machine learning models with well-defined relationships, the database supports reliable storage, efficient querying, and future scalability.
